(Ex)Create a new repository on the command line

echo "# Git_functions_trying" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/Physicsphotonmodel/Git_functions_trying.git
git push -u origin main

這是在創建一個名為 README.md 的檔案，並在裡面寫入 # Git_functions_trying 這行文字（通常作為專案的標題）。
初始化：在目前的資料夾建立一個隱藏的 .git 資料夾，讓 Git 開始追蹤這個目錄下的檔案變動。
提交存檔：將暫存區的變動正式記錄到 Git 的歷史紀錄中。-m 後面接的是這次提交的說明訊息（此處為 "first commit"）。
重新命名分支：將預設的分支名稱（通常是 master）強制重新命名為 main。這是目前 GitHub 推薦的預設主分支名稱。
設定遠端路徑：將這個本地倉庫連結到 GitHub 上的遠端儲存庫。origin 是這串網址的代稱（縮寫）。
上傳並關聯：將本地的 main 分支內容推送到遠端的 origin。
-u 指的是 Upstream（上游），設定後未來只需輸入 git push 或 git pull，Git 就會自動知道要對應到遠端的 main 分支。

(Ex) Add new codes to git

git add .
git commit -m "update message"
git push --set-upstream origin dev4 -->  "dev4" is a branch name

加入所有變動：將目前目錄下「所有」新增、修改或刪除的檔案全部加入暫存區（Staging Area）。最後的句點 . 代表當前路徑。
提交更新：將暫存區中的變動封裝成一個新的節點，並加入開發歷史中。建議在 -m 後的對話框寫下具體的修改內容（例如：fix: button style）。
推送到雲端：將本地端新建立的提交（Commits）上傳到遠端儲存庫（如 GitHub）。由於之前已設定過 -u，這裡不需再輸入分支名稱。

(Ex) Check repository status

git status
git log --oneline

檢查狀態：查看目前哪些檔案已被修改但尚未暫存，以及哪些檔案已在暫存區準備好要提交。這是開發過程中最常使用的確認指令。
檢視簡短歷史：以一行一則的方式列出過去的提交紀錄，讓你快速確認目前的版本位置與編號（Commit ID）。

(Ex) Create and switch to a new branch

git branch dev
git checkout dev
git checkout -b feature-login

建立新分支：創建一個名為 dev 的新分支，但目前仍會停留在原本的分支上。這就像是從當前的進度點拉出一條新的平行線。
切換分支：將工作目錄切換到 dev 分支。接下來你所做的任何修改和提交，都只會記錄在這個分支上，不會影響到 main。
建立並直接切換：這是最常用的快捷指令。它會同時完成「建立分支」與「切換過去」兩個動作。範例中的 feature-login 是分支名稱。

(Ex) Stash (如果你發現帶過來的變動會造成混亂，想先把它「藏起來」，可以使用)

git stash
git stash pop

將目前尚未 commit 的變動暫存到一個秘密口袋。
之後想拿回來時再彈出來。

(Ex) The Standard 3-Step Workflow


git add hello.cpp
git commit -m "feat: add hello world script"
git log --oneline

(git commit --amend -m "") --> 修改上一次的回答

準備檔案：先將檔案加入暫存區。
正式提交：加上 -m 參數並寫下訊息。
確認紀錄：查看是否成功產生了一個新的 Commit。

(Ex) Practical situation 1

error: Your local changes to the following files would be overwritten by checkout:
        Example.txt
    Please commit your changes or stash them before you switch branches.
    Aborting

    代表建立在main之上的dev4不能和main有內容衝突，現在還留在dev4

    --->修改方式
        (1)直接存檔 ==> git add --> git commit -M
        (2)先藏起來不理他 ==> git stash ==> git checkout --> git stash pop
        (3)直接忽略剛剛改過的所有東西 ==> git checkout -- Example.txt 


(Ex) Practical Situation 2 : Understand Git Markers (HEAD, Branches, Remotes)

fa3d653 (HEAD -> dev4) update Example.txt before switching branch
4f6cdb3 (origin/dev4) Add a C++ file in branch dev4
8a5e924 (origin/main, main) this version has update a python code
39d59bc first commit

HEAD 是目前的指針，代表你「現在」正處於哪一個 Commit。
白話文：你現在眼睛看著的、工作目錄裡呈現的程式碼版本，就是 HEAD 所指的地方。
在此範例中，(HEAD -> dev4) 表示你正站在 dev4 分支的最前端。

dev4 / main

本地分支名：這是你電腦上的分支標籤。
分支位置：

dev4 已經走到 fa3d653 這個版本了。
main 還停留在 8a5e924 這個版本。
origin/main / origin/dev4
遠端分支名：origin/ 開頭代表 GitHub 伺服器上的進度。

同步狀況檢查：
origin/main 與你的本地 main 在同一個位置，代表你的 main 分支跟雲端同步。

注意！ 你的本地 dev4 (在 fa3d653) 比 origin/dev4 (在 4f6cdb3) 領先了一個 commit。這代表你剛才做的存檔還沒 push 到 GitHub。
